## Сервис обоев: конфигурация и управление

Этот документ описывает, как работает `WallpaperService` в текущей конфигурации Quickshell, где хранится настройка и какие IPC-команды доступны для управления обоями на отдельных мониторах. Конфигурация читается **один раз при старте**; изменения на диске подхватываются только после перезапуска оболочки.

---

### 1. Структура конфигурации

Рабочий файл конфигурации располагается в `~/.config/quickshell/wallpaper.json` (создаётся автоматически при первом запуске). В дальнейшем все изменения, выполненные через IPC или вручную, сохраняются обратно в этот JSON: сервис использует его как постоянное хранилище текущего состояния. Пример структуры можно посмотреть в репозитории (`config/wallpaper.json`) и при необходимости скопировать в домашний каталог.

```jsonc
{
  "primaryMonitor": "DP-2",                    // монитор, с которого берётся текущий фон для темы
  "defaultWallpaper": "/path/to/image.jpg",   // обои по умолчанию, если нет специфических
  "postSetScript": "",                        // (опционально) путь до исполняемого скрипта-хука
  "global": {
    "directory": "",                          // общий каталог для всех мониторов (опционально)
    "randomOrder": true,                      // глобальный порядок переключения (random/queue)
    "autoChange": {
      "enabled": false,
      "intervalMs": 300000                    // интервал таймера при autoChange
    }
  },
  "monitors": {
    "DP-2": {
      "wallpaper": "",                        // конкретный файл для монитора (если задан)
      "directory": "/home/.../3440x1440",     // каталог со списком обоев (опционально)
      "fillMode": 2                           // режим заполнения (см. ниже)
      // "randomOrder": true                  // индивидуальное поведение для каталога, необязательно
    },
    "DP-1": {
      "wallpaper": "",
      "directory": "/home/.../1080x1920",
      "fillMode": 2
    }
  }
}
```

#### FillMode

Используются стандартные значения `QtQuick/Image.fillMode`:

0 – Stretch  
1 – PreserveAspectFit  
2 – PreserveAspectCrop (по умолчанию)  
3 – Tile  
4 – TileVertically  
5 – TileHorizontally  
6 – Pad

### 2. Логика загрузки и применений

1. **Загрузка конфигурации** – при старте `WallpaperService` читает JSON. Ошибки чтения ведут к восстановлению файла из встроенных дефолтов. Последующие изменения файла на диске не отслеживаются; чтобы применить правки вручную, перезапустите Quickshell.
2. **Текущие состояния** – для каждого монитора сервис хранит:
   - активные обои (`monitorWallpapers`)
   - режим заполнения (`monitorFillModes`)
   - каталог для сканирования (`monitorDirectories`)
   - списки файлов и текущие индексы пролистывания, если каталог задан
   - очередь пост-скриптов, если настроен `postSetScript`
   - последний выбранный файл, фиксируется в конфиге; при следующем старте он используется без смены на случайный
3. **Сканы каталогов** – запускаются по очереди `find` для:
   - общего каталога (если указан)
   - отдельного каталога каждого монитора  
   При успешном скане:
   - первый файл применяется к монитору (если требуется)
   - списки файлов сортируются и кешируются
4. **Таймер** – если `autoChange.enabled == true`, включается `Timer`, который вызывает `nextWallpaper` через заданный интервал.
5. **Fallback** – если для монитора нет каталога/файла, используется `defaultWallpaper` из того же JSON или путь основного монитора. Значение по умолчанию задаётся только в конфигурации, в коде хардкода нет.
6. **Пост-скрипт** – если указан `postSetScript`, после установки обоев запускается внешний скрипт (через `bash -lc`). В него передаются аргументы: `<monitor> <wallpaperPath>`. Для глобальной смены (без конкретного монитора) в качестве имени передаётся пустая строка.

### 3. Использование IPC

Целевой идентификатор IPC: `wallpaper`. Вызовы выполняются через `quickshell -p wallpaper:<команда> ...`.

#### Основные команды

| Команда                                      | Описание                                                                 | Пример |
|----------------------------------------------|---------------------------------------------------------------------------|--------|
| `set <monitor> <path>`                       | Установить конкретный файл на указанный монитор                           | `quickshell -p wallpaper:set DP-2 /path/to/file.jpg` |
| `setAll <path>`                              | Установить файл на все мониторы                                          | `quickshell -p wallpaper:setAll /path/to/file.jpg` |
| `setDirectory <path>`                        | Назначить общий каталог (для всех, если нет индивидуальных)              | `quickshell -p wallpaper:setDirectory /home/user/walls` |
| `setDirectory <monitor> <path>`              | Назначить каталог для конкретного монитора                                | `quickshell -p wallpaper:setDirectory DP-1 /home/user/walls/1080x1920` |
| `next [<monitor>]`                           | Перейти к следующему файлу из каталога (по монитору/всем)                | `quickshell -p wallpaper:next DP-2` |
| `previous [<monitor>]`                       | Перейти к предыдущему файлу                                              | `quickshell -p wallpaper:previous` |
| `setAutoChange <true/false> [<intervalMs>]`  | Включить/выключить автосмену + интервал                                  | `quickshell -p wallpaper:setAutoChange true 600000` |
| `setRandom <true/false>`                     | Режим случайной смены (глобально)                                        | `quickshell -p wallpaper:setRandom false` |
| `setFillMode <monitor> <mode>`               | Выставить fillMode для монитора                                          | `quickshell -p wallpaper:setFillMode DP-2 2` |
| `setFillModeAll <mode>`                      | Выставить fillMode для всех                                               | `quickshell -p wallpaper:setFillModeAll 1` |
| `status`                                     | Показать текущее состояние сервиса (каталоги, обои, настройки)           | `quickshell -p wallpaper:status` |
| `postSetScript`                              | (через конфиг) путь скрипта, вызываемого после установки обоев          | редактируется вручную в JSON |

**Примечания**
* Все аргументы передаются как строки. Сервис сам парсит числовые значения (`parseInt`) или ожидает `"true"/"false"`.
* Если команда принимает необязательный монитор, можно вызвать без него: `next` ⇢ пролистывание по всем мониторам с каталогами.
* При вызове `setDirectory` с одним аргументом каталог считается глобальным. Второй аргумент обязателен для монитор-специфичного каталога.
* После любой команды конфигурация пересохраняется в `~/.config/quickshell/wallpaper.json`. При ручном редактировании обязательно перезапустите Quickshell, чтобы сервис перечитал файл.

### 4. Конвейер смены обоев

1. **Каталоги** – `find` собирает список файлов (расширения: jpg/jpeg/png/webp). Списки сортируются, но при `randomOrder == true` выбор происходит случайно.
2. **Переключение** – `nextWallpaper` / `previousWallpaper` для каждого монитора обновляют индекс. При комбинированной конфигурации:
   - монитор со своим каталогом использует его список;
   - монитор без своего каталога, но при глобальном каталоге, использует глобальный список;
   - монитор без каталогов всегда остаётся на конкретном файле (`wallpaper`).
3. **Автосмена** – если включена, каждая итерация вызывает `nextWallpaper` без указания монитора, тем самым листая все наборы.
4. **Хуки** – после успешной установки фонового изображения в очередь ставится выполнение `postSetScript`. Команда запускается последовательно, поэтому длительные скрипты блокируют последующие вызовы. Скрипт обязан быть исполняемым; вместо абсолютного пути можно указать любой shell-команду, поскольку вызов идёт через `bash -lc '…'`.

### 5. Практическое руководство

1. **Базовая настройка**  
   Отредактируйте `~/.config/quickshell/wallpaper.json` или вызовите:
   ```bash
   quickshell -p wallpaper:setDirectory DP-2 /home/you/wallpapers/3440x1440
   quickshell -p wallpaper:setDirectory DP-1 /home/you/wallpapers/1080x1920
   quickshell -p wallpaper:setFillMode DP-1 1
   ```
   После выполнения сервис пересохранит JSON.

2. **Разовая смена**  
   ```bash
   quickshell -p wallpaper:set DP-1 /path/to/vertical.png
   ```

3. **Каталог и автосмена**  
```bash
quickshell -p wallpaper:setDirectory /home/you/wallpapers/shared
quickshell -p wallpaper:setRandom true
quickshell -p wallpaper:setAutoChange true 300000   # каждые 5 минут
```

5. **Пост-обработка обоев**
   ```bash
   # В ~/.config/quickshell/wallpaper.json:
   "postSetScript": "/home/you/bin/apply-theme.sh"
   ```
   Скрипт получит аргументы `"$1" = MONITOR`, `"$2" = /path/to/wallpaper`. Для глобальной смены `MONITOR` придёт пустой строкой.

4. **Проверка состояния**  
   ```bash
   quickshell -p wallpaper:status
   ```
   В ответе будут перечислены мониторы, текущие файлы, директории и глобальные флаги.

### 6. Диагностика

* **Каталог пуст / нет изображений** – команда `find` вернёт пустой результат. Сервис тогда не обновит обои и выведет предупреждение в логи. Проверить список можно через `WallpaperService.monitorFiles`.
* **Путь к обоям некорректен** – Image в `Wallpaper.qml` выведет ошибку вида `Failed to load wallpaper for DP-1`. Проверьте права доступа и корректность пути.
* **IPC не срабатывает** – убедитесь, что target `"wallpaper"` зарегистрирован (`quickshell -p wallpaper:status`). Также проверьте синтаксис команды (`quickshell -p wallpaper:set …`).
* **Конфигурация не сохраняется** – возможно, JSON невалидный (например, ручное редактирование с синтаксической ошибкой). Сервис восстановит файл из дефолта и сообщит в консоль. Помните: изменения применяются только после рестарта.
* **Пост-скрипт не запускается** – проверьте, что `postSetScript` указывает на исполняемый файл или корректную команду для `bash -lc`. Смотрите логи на сообщения `Wallpaper post-script ...`.

---

### 7. Расширение

* **Дополнительные параметры** – можно добавить свои поля в `monitors[*]`, сохранив актуальные ключи. Сервис игнорирует неизвестные поля, но возвращает их в JSON при сохранении.
* **Обновление логики сканирования** – реализация использует последовательный запуск `find`. Чтобы добавить фильтры или изменить глубину обхода, правьте `scanProcess.command`.
* **Новые команды** – допишите функции в `IpcHandler` с явным указанием типов (возвращаемое значение и аргументы). Помните: максимум 10 аргументов, только типы, перечисленные в документации (string/int/bool/real/color).

### 8. Быстрый чек-лист

1. JSON в `~/.config/quickshell/wallpaper.json` корректен? (`jq . ~/.config/quickshell/wallpaper.json`)
2. Директории существуют и содержат изображения?
3. Команда `quickshell -p wallpaper:status` показывает ожидаемые каталоги и fillMode?
4. Обои переключаются без ошибок (смотрите логи Quickshell)?
5. Не забыли обновить fillMode при смене ориентации?

При соблюдении этих пунктов `WallpaperService` обеспечивает гибкое управление обоями для каждого монитора через конфигурацию и удобные IPC-команды.
